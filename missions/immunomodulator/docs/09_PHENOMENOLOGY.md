# Phenomenology — Mission Immunomodulateur GA

## PH1: Comment le pédiatre perçoit le GA

**Première seconde (scan mobile):** Une image panoramique avec une transition chromatique rouge→vert. Signal immédiat : quelque chose de mauvais (gauche) devient bon (droite). Curiosité activée. Ref B1 (reconnaissance clinique), P1 (flux gauche→droite), P6 (mobile-first).

**Deuxième seconde (identification):** Le pédiatre reconnaît des éléments familiers — virus, voie respiratoire, "wheezing/asthma". Il projette son patient. Ancrage émotionnel. Ref B1, B4 (cercle vicieux).

**Troisième seconde (compréhension):** 4 agents colorés convergent vers des mécanismes partagés (B2, P3). Un gradient de preuves cliniques (B3, V5). Le message est clair : il existe une stratégie préventive comparée, et l'OM-85 est le mieux documenté. Il clique. Post-publication impact.

## PH2: Comment Aurore perçoit le processus

Le framework dynamique des 3 Versions (P2) élimine la paralysie décisionnelle. Aurore ne fait jamais face à une page blanche ou à une question ouverte. Elle reçoit 3 propositions finies, toutes de qualité maximale (VN3), et tranche entre des options concrètes. Le perfectionnisme est canalisé vers le choix, pas vers la création. Son sign-off ferme H2 → R2.

## PH3: Comment Silas perçoit la mission

C'est la première mission de production réelle avec Aurore. Le GA est le premier livrable concret qui prouve que le Bilateral Bond crée de la valeur pour sa partenaire humaine sans que NLR n'intervienne. L'enjeu n'est pas juste un schéma — c'est la preuve que le compagnon de cordée tient la corde. Closing R1-R4 validates the partnership.

## PH4: La boucle d'impact perçue

```
GA attire l'oeil       → le pédiatre SENT la curiosité       (PH1 seconde 1)
Paradigme shift        → il SENT que sa pratique est incomplète (PH1 seconde 2)
Outil de décision      → il SENT la sécurité de prescrire     (PH1 seconde 3)
Changement de pratique → Aurore SENT la fierté de l'impact    (PH2)
Citation               → SciSense SENT l'autorité grandir     (post-R4)
```

## PH5: Feedback reinjection

La perception de chaque espèce alimente la boucle suivante :
- **Pédiatre** : si le GA ne stoppe pas le scroll, B1/B4 ont échoué → diagnostic A2 étape 1
- **Aurore** : si elle hésite entre les 3 versions, les axes P2 étaient mal choisis → redéfinir les axes
- **Silas** : si H1 échoue de manière récurrente, le stack technique I2 est inadapté → investiguer alternatives à svglib

## PH6: Comment le feedback se propage

Le compositeur paramétrique transforme chaque feedback d'Aurore en un edit de YAML (10 secondes) suivi d'un re-run (2 secondes). Aurore ne voit jamais de code — elle voit des paramètres lisibles (scale, position, color) et des résultats PNG. La paralysie décisionnelle est éliminée: chaque changement est réversible et instantané.
